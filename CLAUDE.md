# APEX SENS CALIBRATOR — CLAUDE.md

## プロジェクト概要

APEX Legendsのスコープ間感度を統一・比較するためのWebツール。
1倍スコープの感度を基準に、2倍以降の全スコープで「同じゲーム内角速度」になるper-optic scalar値を自動計算する。
プロ選手/配信者の感度バランスをプリセットとして選択し、自分の1倍感度にスケーリングする機能も搭載。

**ターゲット**: APEX LegendsのPAD/マウスプレイヤー（初心者〜上級者）
**言語**: 日本語のみ
**デプロイ先**: 未定（静的サイトとしてVercel/Netlify等を想定）

## コア設計思想

### 1. 入力のシンプルさ

ユーザーが入力するのは基本2つだけ:
- **ゲーム内FOV設定**（70〜120の整数）
- **1倍スコープのscalar値**

PAD/マウスの切り替えは「入力側」ではなく「出力側（cfg変数名）」で分離。
計算ロジック自体はPAD/マウスで同一。

### 2. 計算式の二重構造

**マウス用（power=1.0）**: 完全な角速度統一
```
ratio = (tan(FOV_1x/2) / tan(FOV_scope/2)) ^ 1.0
```

**PAD用（power=0.2がデフォルト）**: エイムアシスト考慮の緩やか補正
```
ratio = (tan(FOV_1x/2) / tan(FOV_scope/2)) ^ 0.2
```

べき乗のpower値はスライダーで0〜1.0の間で調整可能。
この設計の理由は `docs/AIM_ASSIST.md` に詳述。

### 3. プリセットは「比率ベース」

プロ選手の設定は「1倍の値で全体を割った比率」として保存。
ユーザーの1倍感度に比率を掛けてスケーリングする。
これにより、ベース感度が違うユーザーでも同じ「カーブの形」を適用できる。

プリセットの追加方法は `docs/PRESETS.md` に詳述。

## 重要な技術的注意点

### FOVの丸め誤差

ゲーム内FOVスライダーの値と実際の内部FOVにはズレがある。
```
cl_fovScale = 1 + (スライダー値 - 70) × 0.01375
実FOV(4:3) = 70 × cl_fovScale
```
例: FOV「110」→ cl_fovScale=1.55 → 実FOV=108.5°（110°ではない）

ツール内では**ゲーム内の数字をそのまま入力**してもらい、裏でcl_fovScaleに変換して計算する。
ユーザーにcl_fovScaleを入力させない設計。

### スコープの「公称倍率」と「実質倍率」の違い

APEXの「2倍スコープ」はヒップファイア基準の2倍であり、1倍スコープ基準だと約1.65倍。
1倍スコープ自体がヒップファイアから約1.21倍ズームしているため。
詳細は `docs/CALCULATION.md` 参照。

### エイムアシストとper-optic感度の干渉

PADでper-optic値を上げるとエイムアシストのスローダウン効果が相対的に弱まる。
マウスの角速度統一計算をそのままPADに適用すると高倍率が速すぎてアシストが機能しなくなる。
これがべき乗パラメータ（power）を導入した理由。
詳細は `docs/AIM_ASSIST.md` 参照。

### cfg出力の違い（PAD vs マウス）

**PAD**: `gamepad_ads_advanced_sensitivity_scalar_0`〜`_7` + `gamepad_use_per_scope_sensitivity_scalars "1"`
**マウス**: `mouse_zoomed_sensitivity_scalar_0`〜`_6`

PADの `_7` はアイアンサイト用で、プリセットごとに値が異なる（IRON_RATIOS参照）。

### profile.cfg読み取り専用の警告

ゲーム内でALC設定を開くとper-optic値がリセットされるバグが存在。
cfgファイルを読み取り専用にする対策をUIで警告表示している。

## コーディングルール

- 日本語のコメント・ドキュメントを優先
- CSS-in-JSではなくstyleタグ方式（現在の形式を維持）
- 計算ロジックの変更時は必ず `docs/CALCULATION.md` も同時更新
- プリセット追加時は `docs/PRESETS.md` に元データとソースURLを記録
- ゲーム内表記とcfg変数名の対応を間違えないこと（scalar_0=1x, scalar_1=2x, ...）

## デザインルール（スペーシング）

**8の倍数グリッド**をベースとする:
- 通常の余白: 8px / 16px / 24px / 32px / 48px / 64px
- 詰めが必要な箇所のみ: 4px を使用可
- 1px・2px・3px・5px・6px・7px などの中途半端な値は避ける
- 要素間を「詰める」判断が必要な場合は 4px を使い、それ以外は 8px 以上で揃える
