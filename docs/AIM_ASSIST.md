# エイムアシストのリサーチ結果まとめ

## なぜこのドキュメントが重要か

per-optic感度の計算において、マウスとPADで同じ計算式を使うと**PADでは不適切な結果**になる。
エイムアシストが「感度の減算」として作用するため、感度を上げるとアシストの相対効果が薄まる。
この知見がリニアプリセットの「べき乗パラメータ(power)」設計の根拠。

---

## エイムアシストの二重構造

### 1. スローダウン（Slowdown）
- 照準がヒットボックス外側約0.5体分の「バブル」に入ると視点移動が減速
- 「可変感度マルチプライヤ」として機能
- 右スティックだけでも発動

### 2. 回転アシスト（Rotational Aim Assist）
- **左スティックで移動中のみ**発動
- 照準が敵の動きを自動追従。反応速度は事実上0ms
- 近距離戦のトラッキング安定の最大要因

### 存在しないもの
- 弾丸マグネティズム（弾道補正）→ APEXには**ない**
- 自動スナップ → **ない**
- スティック入力ゼロ時のアシスト → **発動しない**

## スコープ倍率ごとのエイムアシスト有効/無効

| スコープ | エイムアシスト | 備考 |
|---------|------------|------|
| アイアンサイト | ✅ 有効 | |
| 1x | ✅ 有効 | PAD最適 |
| 2x | ✅ 有効 | |
| 3x | ✅ 有効 | 体感やや弱い |
| 2-4x可変 | ✅ 両倍率で有効 | |
| **6x以上** | **❌ 完全無効** | スナイパースコープ全般 |

**6倍以上でアシストが完全停止するのはローンチ以来不変の仕様。**

---

## 1〜4倍でもアシスト体感が異なる理由

エイムアシストの内部値（グローバルスカラー）はスコープ倍率で変化しない。
しかし高倍率で弱く「感じる」理由が3つある:

1. **FOVスケーリング係数**で実効感度が下がる（1x=1.0, 2x≈0.9, 3x≈0.75）
2. **画面上の敵の見かけ移動速度が増大**し、固定アシスト強度では追従困難
3. **バブルの相対サイズが縮小**し、バブル内に留まりにくい

---

## per-optic感度とアシストの干渉

**感度を上げる = アシストの相対効果が減る**

例: 3倍スコープの感度を上げて角速度を1xと統一しようとすると（FOVスケーリング係数0.75の逆数=約1.33倍の感度）、スローダウン効果が相対的に弱まる。

これが**マウスの角速度統一計算をPADにそのまま適用してはいけない理由**。

### べき乗パラメータの導入根拠

プロ選手の実測値を逆算すると:
- shiro氏（均等補正型）の1x→4xの上昇率: 約1.18倍
- satuki氏（ジャンプ型）の1x→4xの上昇率: 約1.08倍
- マウスの完全統一では1x→4xの上昇率: 約3.30倍

つまりプロの設定は完全統一(power=1.0)の10〜30%程度しか上げていない。
`ratio = fullRatio ^ power` でpower=0.2にすると、プロの実測値域に近い出力が得られる。

## 「数字感度」vs「詳細感度(ALC)」のアシスト差異問題

日本コミュニティの定説:
- **数字感度**: 敵と重なった瞬間にスローダウン発動 → 吸い付き強い
- **詳細感度**: 敵の手前からスローダウン開始 → ブレーキ的で弱く感じる

ただし一部の検証者（shiro氏）は差を再現できなかったと報告。パッチで変更された可能性あり。

**本ツールはハイブリッド感度（数字感度 + per-opticスコープ設定のみALC適用）を前提としているため、数字感度のアシスト特性が維持される。**

## 結論: ツール設計への反映

1. リニアモードはデフォルトpower=0.2（PAD向き）にし、スライダーで調整可能に
2. プロ選手の比率プリセットを提供し、数学的計算の代替として選択可能に
3. 高倍率スコープの注意喚起（6倍以上でアシスト無効、感度上げすぎ注意）
4. マウスユーザーはpower=1.0で完全統一が適切
